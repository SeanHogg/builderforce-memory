What this does

state-spaces/mamba models use the same Mamba SSM architecture as mambacode.js. This converter reads their .safetensors weights and writes them in the MBJS binary format that mambacode.js can load.

Qwen2.5-Coder is a Transformer — different architecture, cannot be converted. The only part of Qwen used in mambacode.js / MambaKit is the tokenizer vocabulary.
Vocabulary mismatch: state-spaces/mamba was trained with the GPT-NeoX tokenizer (50,257 tokens). mambacode.js uses Qwen2.5-Coder (151,936 tokens). The SSM layers transfer cleanly; the embedding table is padded with initialised values for the extra Qwen tokens. Run a few passes through pretrain.html after converting to warm up the new token slots.
